# Release v2.0.0

Fecha: 2026-04-30

## Resumen

Esta versión incorpora un sistema avanzado de márgenes por categoría en recetas comerciales con cálculo de totales por moneda, soporte de permisos basados en cargo, mejoras en tiempo real (SSE) para notificación de cambios de precio, y refinamientos en exportación PDF con lógica de permisos.

## Cambios principales

### Módulo de Márgenes por Categoría en Recetas

- Se implementó modal `info-categoria-modal` para edición interactiva de márgenes por `sub_cat_1` (subcategoría principal).
- Características:
  - Tabla con columnas: `Categoría`, `Cantidad`, `Subtotal`, `Margen %`, `Total Fórmula`.
  - Cálculo de `Total Fórmula` usando: `subtotal / (1 - margen_decimal)`.
  - Inputs `+/-` para incrementar/decrementar margen de forma granular.
  - Resumen de totales por moneda: `Total S/.` y `Total $`.
  - Normalización de etiquetas: evita mostrar duplicados como "X (X)" → muestra solo "X".
  - Guardado automático de márgenes, cantidad y subtotal en `receta_categoria`.

### Base de datos: Soporte de Moneda en Categorías

- Se agregó columna `moneda` en tabla `receta_categoria` para mantener totales desagregados por moneda (S/ o $).
- Archivo de migración: `db/migrations/2026-04-30_add_moneda_to_receta_categoria.sql`.
  - Ejecutar esta migración antes de desplegar la versión.

### Permisos por Cargo (Técnico)

- Se implementó sistema de permisos basado en `session_cargo`:
  - `cargo == 4` (Técnico): oculta columna `Precio` y totales sensibles en:
    - Formulario de agregar/editar ítems (`add_item_receta.php`, `upd_item_receta.php`).
    - Listado de ítems (`items_receta.php` con lógica en `assets/js/table-gridjs-item-receta.js`).
    - Modal de categorías (`receta_form.php`).
    - Tabla de composición de receta (`receta.php` con `assets/js/form-receta.js`).
    - PDF de receta (`pdf_receta.php`).
  - El botón `info-categoria-modal` queda deshabilitado para Técnicos.
  - Validación en frontend (via `data-user-cargo`) y backend (`session_cargo`).

### Tiempo Real (SSE): Nuevas Funcionalidades

- **Nuevo endpoint**: `controller/stream_precio_0.php`
  - Emite eventos SSE cuando hay ítems con `precio = 0` en catálogo.
  - Integración en `assets/js/table-gridjs-item-receta.js`.
  - Frontend muestra alerta en div `#alertPrecioCambio` y notificación tipo alertify.
  - Auto-actualización sin refrescar página.

- **Mejoras en stream existente** (`controller/stream_receta_cambios.php`):
  - Firma/checksum optimizado: evita traer datos cuando no hay cambios.
  - Polling adaptativo: modo activo (intervalo corto) vs modo idle (intervalo mayor).
  - Heartbeat controlado para mantener conexión sin exceso de tráfico.
  - Eventos claros: `price_changes`, `ping`, `stream_disabled`, `error`.

### Eliminación de JavaScript Inline

- Se removió JavaScript inline de vistas PHP.
- Datos del servidor ahora pasan mediante atributos `data-*`:
  - Ejemplo: `data-user-cargo` en contenedores para leer cargo en `assets/js/*.js`.
- Archivos JS dedicados:
  - `assets/js/receta_form.js` (manejo de márgenes, tooltips, totales).
  - `assets/js/form-receta.js` (composición de receta, modal items).
  - `assets/js/table-gridjs-item-receta.js` (listado items con SSE precio_0).
  - `assets/js/table-gridjs-receta.js` (listado recetas con tooltip PDF).

### Mejoras en Exportación PDF

- Archivo: `pdf_receta.php`
- Cambios:
  - Logo incrustado como data URI (compatible con Dompdf).
  - Totales por moneda: `Total S/`, `Total $`.
  - `Total Perú` (conversión con tipo de cambio SUNAT).
  - `Total Margen S/`, `Total Margen $`, `Total Margen Perú`.
  - Condicionamiento por permisos: cuando `cargo == 4`, oculta:
    - Columna `Precio` en tabla de detalle.
    - `Total Producto`, `Total Servicio`, totales monetarios.
    - Se muestra solo `Total Items`.
  - Pie de página fijo con tipo de cambio, usuario y fecha/hora de impresión.
  - HTML/CSS compatible con Dompdf (evita reglas no soportadas como `position: fixed`).

## Reglas de negocio aplicadas

1. **Permisos por cargo**:
   - Técnico (`cargo == 4`) no puede ver precios ni acceder a márgenes.
   - Restricción aplicada en frontend y backend.

2. **Márgenes**:
   - Rango válido: 0% a 100%.
   - Fórmula: `Total Fórmula = Subtotal / (1 - margen_decimal)`.
   - Se valida que `margen_decimal < 1` antes de dividir.

3. **Tiempo real**:
   - Stream SSE libera lock de sesión (`session_write_close()`) para evitar bloqueos.
   - Solo activo cuando receta está en estado `Enviada`.
   - Detección de cambios mediante firma/checksum antes de traer datos.

4. **Migraciones**:
   - Se debe ejecutar `db/migrations/2026-04-30_add_moneda_to_receta_categoria.sql` antes de usar la aplicación.

## Endpoints nuevos

- `controller/get_receta_categoria.php` (GET): obtiene categorías con márgenes guardados.
- `controller/upd_receta_categoria.php` (POST): guarda márgenes por categoría.
- `controller/stream_precio_0.php` (GET/SSE): emite notificaciones de ítems con precio = 0.
